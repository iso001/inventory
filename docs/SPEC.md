# 在庫管理システム 仕様書

## 1. システム概要

中間業者の製品倉庫を管理するWebアプリケーション。
複数の仕入先から製品を入庫し、納入先へのオーダー単位で出荷を管理する。

**目的**: 学習用
**想定ユーザー数**: 5名程度
**インフラ**: Vercel (フロントエンド) + Supabase (DB・認証)

---

## 6. ビジネスルール

- 在庫数はシステムが自動管理する（手動変更不可）
- 入庫確定 → 在庫数が増加
- オーダーステータスが `shipped` に変更 → 在庫数が減算
- 在庫数がマイナスになる出荷は **警告を表示する**（ブロックはしない）
- オーダーのキャンセルは `shipped` になる前のみ可能
- `shipped` のオーダーはキャンセル・編集不可

---

## 7. 非機能要件

- Supabase 無料枠内での運用（DB 500MB、同時接続 60）
- Row Level Security (RLS) によるデータアクセス制御
- レスポンシブデザイン対応（モバイル閲覧可能）
